## 詰まったこと（と解決）

| 詰まり | 解決 |
| --- | --- |
| **pnpm の PATH shadowing 警告** | システム環境変数の Path で `C:\Users\sakan\AppData\Local\Volta\bin` を最上段に移動 |
| **`package.json` の JSON エラー**（`scripts` ブロックの `}` の後にカンマが抜けていた） | カンマを追加 |
| **smoke test がデフォルト reporter で 0 passed (1) に見える** | 実は通っていた。`pnpm test --reporter=verbose` で確認すると 2 passed と出る |
| **smoke test の中身が文字化け**（VS Code が Shift-JIS で保存していた） | テスト名を英語に書き換え＋PowerShell の `[System.Text.UTF8Encoding] $false` で UTF-8（BOM なし）で書き直し |
| **`pnpm build` で `tsc -b` が test プロパティ型エラー** | `vite.config.ts` の import を `"vite"` → `"vitest/config"` に変更 |
| **`pnpm lint` で `eslintConfigPrettier is not defined`** | `eslint.config.js` の冒頭に `import eslintConfigPrettier from 'eslint-config-prettier'` を追加し忘れていた |
| **Prettier で 6ファイル style issues** | `pnpm format` で一括整形 |
| **outputs フォルダに PowerShell から到達できない**（Cowork のサンドボックス領域） | `request_cowork_directory` で Claude がプロジェクトフォルダに直接書ける体制に切り替え |
| **`docs/m1_checlist.md` というタイプミスファイル**（手動コピペ時の誤り） | `Remove-Item` で削除 |

### 学び
- **VS Code の保存エンコード**：日本語を含むファイルは右下のステータスバーで `UTF-8` を必ず確認する。`files.encoding: "utf8"` を settings.json に入れておくと安全。
- **Vite/Vitest の型問題**：`vite.config.ts` で `test` プロパティを使うときは `vitest/config` の `defineConfig` を使う。
- **bleeding edge の罠**：scaffolder が入れた Vite 8 / TS 6 / React 19 / Tailwind v4 は最新すぎて手順書（v3/v5/18 想定）と差異が出る。設定が違うところは都度調整。
- **コピペの落とし穴**：JSON の `scripts` ブロック貼り替えではカンマ位置に細心の注意。VS Code の Problems パネルを保存前に確認する。
