## 0. このドキュメントの目的

M3-G の §1.10 と §3 で求められる「M3 完了の手動検証」を、誰でも 1 周で
回せる順序で並べたもの。所要時間の目安は **20〜30 分**。

各項目に「操作」「期待値」「失敗時の対処」を書く。期待値どおりに動けば
チェックを入れていき、すべて埋まれば M3 完了。

実行前に：

```powershell
cd C:\Users\sakan\dev\gambit-architects
pnpm test --run    # まず自動テストが全 PASS であること
pnpm build         # 型・ビルドエラーがないこと
pnpm dev           # ローカル起動
```

`pnpm dev` のローカル URL（通常 `http://localhost:5173`）を Chrome / Edge で開く。

## 1. 編成画面：装備が見える（チェックリスト §3-1）

**操作**

1. ホーム画面の「編成」ボタンをクリック
2. 4 キャラクターのカードを確認

**期待値**

- [ ] 4 枚のキャラクターカードが表示される（Sword1, Sword2, Mage1, Healer1）
- [ ] 各カードに「武器 / 防具 / センサー」の装備サマリ行がある
- [ ] 初期状態は全員「装備なし」または既定装備（空欄表示）

**失敗時**

- カードが 4 枚出ない → `src/ui/screens/PartyScreen.tsx` の party 列描画を確認
- 装備行が無い → EquipmentPicker 統合がしていないか確認

---

## 2. 装備変更モーダルが動く（チェックリスト §3-2、§3-3）

**操作**

1. Sword1 カードの「装備」ボタンをクリック
2. モーダルが開く
3. 武器ドロップダウンを 1 回開いて閉じる（CSS overflow バグ対策の確認）
4. もう 1 回開いて `STEEL_SWORD` を選択
5. 防具ドロップダウンで `PLATE_MAIL` を選択
6. センサードロップダウンで `HP_SCANNER` を選択
7. 「保存」をクリック
8. カードの装備サマリが更新されていること

**期待値**

- [ ] モーダルが開く
- [ ] 武器プルダウンは **2 回目以降も開く**（M3-D で修正済み、回帰）
- [ ] ジョブ制限が効いている：剣士のドロップダウンに `MAGE_ROD` 等が出ない
- [ ] 装備プレビューが「base → 装備後」表示で ATK 25→40 など見える
- [ ] 保存後、カードに「STEEL_SWORD / PLATE_MAIL / HP_SCANNER」のサマリが出る

**失敗時**

- プルダウンが 2 回目以降開かない → `EquipmentPicker.tsx` から `max-h-[90vh]` または `overflow-auto` が再追加されていないか確認

**全員に装備をつける**：Sword2 も `STEEL_SWORD/PLATE_MAIL/STATUS_DETECTOR`、Mage1 は `CRYSTAL_STAFF/ARCHWIZARD_ROBE/ELEMENT_ANALYZER`、Healer1 は `ANCIENT_MACE/CARDINAL_GARB/BASIC_SCANNER` を設定。

---

## 3. ガンビット編集：sensor の効果を組む（チェックリスト §3-4）

**操作**

1. Mage1 カードの「ガンビット編集」をクリック
2. ルール 1：`ENEMY_WEAK_TO(FIRE)` → `ENEMY_MATCH` → `CAST_OFFENSE(FIRA)` を作る
3. ルール 2：`ENEMY_EXISTS` → `ENEMY_MATCH` → `ATTACK`（フォールバック）
4. 保存

**期待値**

- [ ] RulePicker モーダルで上記の組み合わせが選択可能
- [ ] 保存ボタンが有効化される
- [ ] 保存後にカードのルール一覧に 2 行表示される

**期待値（追加）**

- [ ] ルール追加モーダルの条件・対象・行動の **プルダウンが 2 回目以降も開く**（M3-G-6 で修正済み、回帰）

**失敗時**

- 条件・対象・行動が見つからない → DSL v0.3 仕様と RulePicker のリスト同期を確認
- プルダウンが 2 回目以降開かない → `RulePicker.tsx` から `max-h-[90vh]` または `overflow-auto` が再追加されていないか確認（EquipmentPicker と同じ Chromium 仕様の罠）

---

## 4. センサー有り → 確実に弱点突き（チェックリスト §3-4）

**操作**

1. 編成画面に戻る
2. ホーム → 「出撃」ボタンで深度 1 戦闘へ
3. 戦闘ログを目視（or 「スキップ」を押して結果ログを一括表示）

**期待値**

- [ ] GOBLIN（弱点 FIRE）に対して Mage1 が `CAST_OFFENSE(FIRA)` を **毎ターン狙う**
- [ ] ログに「Mage1 → CAST_OFFENSE (target...)」の行が見える
- [ ] 勝利後、深度が 2 に進む
- [ ] 勝利ダイアログに「深度 N+1 へ出撃」と「ホームへ戻る」の 2 ボタンが並ぶ（M3-G-7 で追加）
- [ ] 「深度 N+1 へ出撃」を押すと、ホームへ戻らずそのまま次戦が始まる
- [ ] 敗北時は「ホームへ戻る」だけが出る（次の深度ボタンは非表示）

**失敗時**

- Mage1 が ATTACK しかしない → ELEMENT_ANALYZER が装着されていないか、ENEMY_WEAK_TO 評価が失敗
- 評価器の senseCheck で、装備が読めていない可能性 → `src/gambit/evaluator.ts` の `actorHasSensorFor` 周辺
- 「次の深度へ出撃」ボタンが押せない / 反応しない → `BattleScreen.tsx` の `nextDepth()` で `setRevealedCount(0)` と `recordedRef.current = false` のリセットを確認

## 5. センサーなし → たまに空振り（チェックリスト §3-5）

**操作**

1. 編成画面で Mage1 の装備モーダルを開き、`sensor` を空（解除）に変更
2. 保存
3. 深度 2 で出撃（直前の戦闘で勝てば深度 2 になっているはず）
4. 戦闘ログを目視

**期待値**

- [ ] Mage1 の `CAST_OFFENSE(FIRA)` が **時々ルール 1 をスキップして** ATTACK にフォールスルーする
- [ ] 同じ深度に再出撃すると別の挙動が見られる場合がある（Math.random の影響）

**注意**：M3-E 設計上、センサー無し条件は 50% で失敗→次ルールへ。`Math.random` が均等に分布するので 3〜5 ターン観察すれば差が見える。

---

## 6. PROVOKE タンク動作（チェックリスト §3-6）

**操作**

1. 編成画面で Mage1 の sensor を `ELEMENT_ANALYZER` に戻す（弱点突きを復活させる）
2. 深度 5 まで進む（深度 1〜4 を出撃で順に勝ち抜く）
3. 深度 5 ボス戦（GOBLIN_KING）に出撃
4. 戦闘ログを目視

**期待値**

- [ ] Sword1（presetTank）が初手で `PROVOKE` を打つ（`BOSS_PRESENT` が真）
- [ ] その後 3 ターン、敵 ATTACK が Sword1 にリダイレクトされる
- [ ] Sword2 / Mage1 / Healer1 は敵の物理攻撃を受けない（または受けにくい）
- [ ] 勝利

**失敗時**

- Sword1 が PROVOKE しない → presetTank の r3 と BOSS_PRESENT を確認

## 7. depth 10 ボス：装備込みで勝てる（チェックリスト §3-7）

**操作**

1. 深度 10 まで進む
2. ボス戦（DARK_DRAGON）に出撃
3. スキップで結果まで早送り

**期待値**

- [ ] 勝利（標準パーティ＋装備なら balance.test.ts でも勝つ前提）
- [ ] Healer1 が CAST_HEAL を発動し、味方の HP が回復するイベントが見える
- [ ] Mage1 が HOLY_BOLT または ATTACK を撃つ（DARK_DRAGON は HOLY 弱点）

**失敗時**

- 負ける → balance.test.ts の depth 10 結果と比較。Mage1/Healer1 のガンビットに HOLY_BOLT ルールが入っていないと厳しい

**操作**

1. ホーム → 「ログ」をクリック

**期待値**

- [ ] 直近 5 戦が新しい順に並んでいる
- [ ] 各行に「深度・勝敗・ターン数」が見える
- [ ] depth 10 のボス勝利が記録されている

**失敗時**

## 8.5 言語切替（M3-G-8）

**操作**

1. 画面右上の言語トグル「JA / EN」を確認
2. EN をクリック
3. ホーム、編成、戦闘、ガンビット編集の各画面に移動して表示が英語になっているか確認
4. JA に戻して日本語に戻ることを確認
5. EN 状態でブラウザをリロード → 引き続き英語表示であること（永続化）

**期待値**

- [ ] 右上に「JA / EN」のトグルが見える
- [ ] EN クリックで全画面の固定テキスト・ジョブ名・敵名・装備名・状態名・戦闘ログが英語に切り替わる
- [ ] JA に戻すと日本語に戻る
- [ ] リロード後も最後に選んだ言語が維持される

**失敗時**

- 一部画面だけ日本語のまま → その画面が `useT()` / `localized*()` を使っていない箇所がないか調査
- リロードで戻らない → `storage.ts` のマイグレーション（locale の保存）を確認
