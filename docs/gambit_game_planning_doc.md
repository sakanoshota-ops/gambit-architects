## ― ガンビットでパーティを"プログラム"する個人開発サンドボックス ―

| 項目 | 内容 |
| --- | --- |
| プロジェクト名（仮） | Gambit Architects |
| ジャンル | サンドボックス × オートバトル（ガンビット型AIプログラミング） |
| 開発体制 | **個人開発 + AI支援（Claude等）** |

## 0. スケールダウンの方針

前版（v0.1）からの主な削ぎ落とし：

| 削ったもの | 理由 |
| --- | --- |
| ストーリーモード／世界観の物語 | テキスト・キャラ・固有ボイスの制作コストが個人では重すぎる |
| ネイティブiOS/Android配信 | ストア審査・課金審査・端末検証が個人運用の負荷を跳ね上げる |
| マルチプレイ／コミュニティ共有サーバ | バックエンド運用コストとモデレーション負担を回避 |
| 課金・バトルパス | 収益化に伴う法務・税務・運用コストを回避（投げ銭のみ） |
| ジョブ6種＋サブジョブ | コンテンツ量を絞り、**ジョブ3〜4種**に圧縮 |
| 章構成のキャンペーン | **手作業の章を捨て、procgen（手続き生成）ダンジョン**で量を確保 |
| フルボイス／凝った演出 | 効果音と最低限のSEのみ。アートも記号的・幾何的に統一 |

**残すコア**：ガンビット編集 → 自動戦闘 → ログ観察 → 編集の改善ループ。これだけは絶対に磨く。

---

## 1. コンセプト（変更なし）

### 1.1 ワンライナー
> **「キャラを動かすのではなく、"判断"そのものを設計する。」**

プレイヤーはAIアーキテクトとして、自分のパーティ（オートマタ4体）が自律的に戦う様を組み立てる。

### 1.2 個人開発で勝つための差別化
- 大手と物量で戦わない。**ガンビットの"組む楽しさ"一点突破**。
- グラフィックは"記号化"してでもAIの可視化（誰が・なぜ・何をしたか）に寄せる。
- "クリアする"より"組み立てて眺める"が主。**RPGではなく"AI玩具"**として位置付ける。

## 3. コアループ（変更なし・縮小）

```
┌────────────────────────────────────┐
│ ① 出撃ステージを選ぶ（procgenダンジョン）│
│        ↓                             │
│ ② 自動戦闘（2〜5分・倍速可・スキップ可）│
│        ↓                             │
│ ③ ログ確認（誰がどのルールで何をしたか）│
│        ↓                             │
│ ④ ガンビット編集／装備調整              │
│        ↓                             │
│ ⑤ 再挑戦 or 次の階層へ                  │
└────────────────────────────────────┘
```

**手作業のシナリオ章は廃止**。procgenダンジョンの"深度"が進行ゲージを兼ねる（roguelite風）。

## 4. ガンビットシステム（本作の核）

### 4.1 基本構造（不変）
1キャラごとに、**優先度順に並んだルールリスト**：

```
  [条件]  →  [対象]  →  [行動]
  例：味方HP < 40%  →  最も傷ついた味方  →  ケアル
```

戦闘中、上から評価して最初に成立したルールを実行。

### 4.2 v1.0スコープ（個人開発で組み切れる量）

| カテゴリ | v1.0で実装する量 | v0.1版（フル仕様） |
| --- | --- | --- |
| 条件 | **21種**（DSL仕様 v0.3 で確定） | 100種以上 |
| 行動 | **15種** | 制限なし |
| 対象 | 6種（自分／味方MATCH／味方HP最低／味方全員／敵MATCH／敵全員） | 12種以上 |
| 論理結合 | **AND のみ（v1.0）**、OR は v1.1 | AND/OR 2段 |
| ルール上限 | 1キャラあたり **8スロット** | 12〜16 |

これでも組み合わせは数千通り以上に達し、コア体験は十分成立する。

### 4.3 編集UI
- **縦リスト型のみ**（ノードベースは初版では作らない）。
- スマホでもドラッグで並べ替え可能。
- 条件・行動はアイコンで視認、長押しで詳細説明。

### 4.4 共有
- **サーバ不要のテキスト/QRコード共有**。ガンビットセットを文字列にエクスポート → コピペ／QR読み取りでインポート。
- 公式の"おすすめプリセット"を数本同梱して学習導線にする。

## 5. ゲーム内容（最小限）

### 5.1 オートマタ（プレイアブル）
- **3ジョブ**：剣士（前衛物理）／魔導士（範囲魔法）／治癒士（回復・補助）
- + v1.1以降に追加候補：狩人（弓・状態異常）

各ジョブは初期4体ぶん用意し、編成は**4体固定パーティ**。育成要素はレベル＋装備のみで、複雑なスキルツリーは入れない。

### 5.2 装備
- **武器・防具・センサー**の3スロット。
- センサーは「敵HPの正確な値が見える」「ステータス異常を検知できる」など、**ガンビット条件の精度を上げる**役割。装備とルールの相性を考えるのが本作らしい遊び。

### 5.3 敵・ダンジョン
- **procgenダンジョン**を1種類用意。深度が進むほど敵が強く、特殊属性が増える。
- 敵テンプレートは **15〜20体程度** で、組み合わせとステータス変動で多様性を出す。
- ボスは深度10ごとに1体、合計4〜5種程度。

### 5.4 進行
- 深度を進めると、**新しい条件・行動・装備・敵**がアンロックされる（roguelite的恒常進行）。
- 1ラン死亡してもガンビット設計は失われない。**ルールが資産**になる。

## 7. 技術選定（個人開発に最適化）

### 7.1 推奨スタック

| レイヤ | 採用案 | 理由 |
| --- | --- | --- |
| 言語 | **TypeScript 6.x** | AIコード生成精度が最高、型で品質担保 |
| UIフレームワーク | **React 19** | 本作の主体である UI／状態遷移と最相性 |
| ビルドツール | **Vite 8** | 起動・HMR が速い |
| スタイル | **Tailwind v4**（`@tailwindcss/vite` プラグイン方式） | モバイル UI を素早く組める |
| テスト | **Vitest 4 + Testing Library + jsdom** | ガンビット評価器の単体テストに最適 |
| Lint/Format | **ESLint 10（flat config）+ Prettier 3** | 整形・静的解析を自動化 |
| パッケージ管理 | **pnpm**（Volta 経由） | 高速・省ディスク |
| PWA | **vite-plugin-pwa** | 数行で PWA 化可（M1 では未使用、後で有効化） |
| ビルド先 | **Web（HTML5）** → モバイルブラウザ → PWA | アプリストア審査を回避、配信が早い |
| 永続化 | **localStorage / IndexedDB** | サーバ不要、個人運用で破綻しない |
| アセット | フリー素材（OpenGameArt等）＋AI画像生成、SEはfreesound | 個人で揃えられる |
| ホスティング | **Cloudflare Pages** | 無料、PR プレビュー有り |

### 7.2 採用しないもの
- 専用バックエンド、リアルタイム同期、データベース、ランキングサーバ
- ネイティブ書き出し（v1.0時点）、ストア配信
- 課金SDK、広告SDK、解析SDK（プライバシーポリシーの整備を回避）

### 7.3 AI支援の使い方
- **コード生成**：戦闘ループ、ガンビット評価器、UI雛形
- **アセット生成**：アイコン、背景パターン、キャラのシルエット
- **コンテンツ生成**：敵テンプレート、フレーバーテキスト、プリセットガンビット
- **テスト補助**：ガンビット評価器のユニットテスト網羅

ただし**ゲームデザインの判断（何を残し何を捨てるか）はAIに委ねない**。判断は自分でする。

## 9. リスクと個人開発での対策

| リスク | 対策 |
| --- | --- |
| **モチベ低下で停止** | 月単位で「遊べる状態」を死守。完璧主義より公開を優先。 |
| **スコープが膨らむ** | この企画書を"凍結された憲法"として扱い、追加は v1.1 以降に回す。 |
| **ガンビットUIが操作しづらい** | M2終了時に最低3人にプレイテスト依頼（友人・SNS募集）。 |
| **作っても誰も遊ばない** | 開発過程をX/devlog/itchのDevlog等で発信し、リリース時に動かすコア層を作っておく。 |
| **アート不足で見た目が貧弱** | 記号的・幾何的なミニマルアートに統一し、"そういう作風"として成立させる。 |
| **AI生成コードのバグ蓄積** | ガンビット評価器など中核ロジックには**型と単体テスト**を厳格に書く。AI出力をそのまま信じない。 |

## 10. v1.0で"作らない"ことリスト（重要）

これらは v1.1以降にしか触らない。途中で誘惑されても入れない。

- 多人数共有サーバ／フレンド機能
- 課金・広告
- 凝ったストーリー、立ち絵、ボイス
- ジョブ4種以上
- ノードベース編集UI
- マルチプラットフォーム同時対応
- ローカライズ（v1.0は日本語＋英語まで。それ以上はやらない）
