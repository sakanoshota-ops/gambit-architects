## 0. M2 の目的

> **「友人に渡して触ってもらえる」状態にする**

具体的には、UI で以下のループが完結すること：

```
ホーム → 出撃 → 自動戦闘を眺める → ログ確認 → ガンビット編集 → 再挑戦
```

逆に M2 では**完成度・美しさは追わない**。記号的な見た目で OK、機能が一通り揃うのが目標。

### 2026-05-10 確定の M2 スコープ条件

1. **ターン制継続**：ATB は M3 以降。M1 で動いている turn-based runner を流用
2. **デスクトップブラウザ前提**：スマホ最適化は M5 以降
3. **魔法・スキルの実効果は最小限**：各カテゴリ最低 1 つだけ動かす（FIRE、CURE、RAISE、PROTECT、POISON、ANTIDOTE、POWER_SLASH の 7 つ＋ POISON 状態のターン処理）

## 2. M2 で追加する型・データ

### 2.1 Unit に追加するフィールド
M1 で `defendingThisTurn` を BattleState 側に持たせたのと同様、M2 で必要になる per-unit のフラグ：

```ts
interface Unit {
  // M1 既存...

  // M2 で追加
  /** バフ／デバフの残ターン数（key = Status ID、value = 残ターン）*/
  statusDurations: Partial<Record<Status, number>>;
}
```

POISON は付与時に 5 ターンセット、毎ターン -1、0 でクリア。
PROTECT も同様（4 ターン継続予定）。

### 2.2 ダンジョン進捗
```ts
interface DungeonProgress {
  currentDepth: number;  // 1〜5
  maxDepth: number;      // localStorage に保存。M2 中の最高記録
  lastBattle?: {
    winner: "ALLY" | "ENEMY" | "TIMEOUT";
    turns: number;
    depth: number;
  };
}
```

### 2.3 プレイヤーデータ
```ts
interface PlayerData {
  party: [Unit, Unit, Unit, Unit];   // 4 体固定
  dungeon: DungeonProgress;
  settings: {
    battleSpeed: 1 | 2 | 4;
  };
}
```

すべて localStorage の単一キー `"gambit-architects-save-v1"` に JSON で保存。

## 3. M2 デモ：「これが動けば M2 完了」

`pnpm dev` で localhost を開き、以下が**手で 1 周できる**こと：

1. ホーム画面が出る。深度 1
2. 「出撃」を押す → 戦闘画面に遷移、自動進行
3. 4 剣士 vs ゴブリン 2 体 → ALLY 勝利、深度 2 にアップ
4. ホームに戻る → 編成 → 1 体を「魔導士」に変更
5. ガンビット編集 → r2 を `ENEMY_WEAK_TO(FIRE) → ENEMY_MATCH → CAST_OFFENSE(FIRE)` に編集 → 保存
6. ホーム → 出撃 → 戦闘画面で「魔導士が FIRE を撃つ」が見える
7. ログ画面 → 直近の戦闘ログが残っている
8. 設定 → ガンビットを export → 文字列をコピー → import で復元できる

この一連が手で完走できれば M2 完了。

## 5. M2 で発生しがちな罠と回避策

| 罠 | 対策 |
| --- | --- |
| **UI の見た目に時間を溶かす** | M2 は機能が一通り動けば OK。Tailwind デフォルトの組み合わせで十分。配色は1パレット |
| **ガンビット編集 UI を完璧にする** | 最初は 3 ステップピッカーが動けば OK。ドラッグ＆ドロップは「↑↓」ボタンで代用してもよい |
| **戦闘演出に凝る** | M2 はテキストとフェードだけ。スプライト・アニメは M5 で |
| **状態管理を Redux 系に移行** | Context + useReducer で十分。複雑化したら Zustand。Redux は v1.1 以降の検討事項 |
| **テストを書かなくなる** | UI でも最低限の振る舞いテストは書く。E2E までは要らないがコンポーネントテストは必須 |
| **コア（評価器・ランナー）を触る** | **絶対に触らない**。M1 で固めたコアは M2 では動かさない。新機能は別レイヤで追加 |
